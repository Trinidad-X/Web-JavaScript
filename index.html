Git & GitHub

GitHub is a cloud platform for hosting code, allowing multiple developers to work together using Git (Version Control).

Key Concepts:

  • Repository (Repo): The folder where your project and its history lives.
  • Commit: A save point in your history. Like a checkpoint in a game.
  • Push: Uploading your local commits to the Cloud (GitHub).
  • Pull: Downloading changes from GitHub to your computer.
  • Branch: A parallel version of the code to test new features without breaking the main site.
Markdown.md

What is Markdown?

Markdown is a lightweight markup language used to format plain text. It allows writers to add structure—such as headings, lists, links, and code—using simple, readable symbols.

Why Use Markdown?

  • Easy to read and write
  • Works in plain text editors
  • Converts easily to HTML and other formats
  • Widely supported (GitHub, documentation tools, note apps)

Common Markdown Syntax

  • Headings: # Heading
  • Bold: **bold text**
  • Italic: *italic text*
  • Lists: - Item
  • Links: [Text](URL)
  • Code: `inline code`

Where Markdown Is Used

Markdown is commonly used for:

  • README files
  • Technical documentation
  • Blogs and articles
  • Notes and knowledge bases

Summary

Markdown provides a simple way to format text without complex tools. Its simplicity and portability make it a popular choice for developers, writers, and content creators.

What is HTML5?

HTML5 is the latest version of the HyperText Markup Language (HTML), which is used to create and structure content on the web. It introduces new elements, APIs, and features that make web pages more interactive and multimedia-friendly.

Key Features of HTML5

  • New semantic elements: <header>, <footer>, <article>, <section>
  • Built-in multimedia support: <audio>, <video>
  • Graphics and drawing: <canvas>, SVG integration
  • Enhanced forms: new input types and attributes
  • APIs for offline storage, geolocation, drag-and-drop, and more
  • Improved accessibility and SEO through semantic tags

Why Use HTML5?

  • Better structure and semantics for web content
  • Supports multimedia without third-party plugins
  • Compatible with modern browsers and devices
  • Enables richer web applications and interactivity

Common HTML5 Elements

  • <header> - Page or section header
  • <nav> - Navigation links
  • <section> - Thematic grouping of content
  • <article> - Self-contained content
  • <footer> - Footer content
  • <video> and <audio> - Media playback
  • <canvas> - Draw graphics via JavaScript

Summary

HTML5 modernizes web development with semantic structure, multimedia support, and advanced APIs. It is essential for building interactive, accessible, and responsive websites today.

styles.css

What is CSS3?

CSS3 (Cascading Style Sheets Level 3) is the latest version of CSS, the language used to style and layout web pages. CSS3 allows developers to control the look and feel of websites, including colors, fonts, spacing, and responsive design.

Key Features of CSS3

  • New selectors for targeting elements more precisely
  • Advanced backgrounds and borders (gradients, rounded corners, shadows)
  • Flexible box layout (flexbox) and grid layout (CSS Grid)
  • Transitions and animations for smooth effects
  • Media queries for responsive design
  • Custom fonts with @font-face

Why Use CSS3?

  • Enhances visual design and user experience
  • Enables responsive layouts for mobile and desktop
  • Supports animation and interactive effects without JavaScript
  • Improves maintainability by separating content (HTML) from presentation (CSS)

Common CSS3 Properties

  • border-radius – rounded corners
  • box-shadow – element shadows
  • linear-gradient, radial-gradient – background gradients
  • transition – smooth changes in styles
  • transform – rotate, scale, or move elements
  • flex, grid – layout control
  • @media – responsive design rules

Summary

CSS3 is a powerful tool for designing modern web pages. With advanced styling, layout options, and animation features, it allows developers to create visually appealing and responsive websites that adapt to any device.

Data.xml

What is XML?

XML (eXtensible Markup Language) is a markup language designed to store, transport, and organize data in a structured, human-readable, and machine-readable format. Unlike HTML, which focuses on displaying data, XML focuses on describing and structuring data.

Key Features of XML

  • Self-descriptive: uses tags to define the meaning of data
  • Hierarchical structure with nested elements
  • Platform-independent and widely supported for data exchange
  • Extensible: you can create your own custom tags
  • Can be validated using DTD or XML Schema for correctness

Why Use XML?

  • To store and transport structured data across systems
  • To separate data from presentation (unlike HTML)
  • For configuration files, web services (SOAP), and APIs
  • To ensure data is readable by both humans and machines

Common XML Syntax

  • Elements: <name>value</name>
  • Attributes: <person age="30">John</person>
  • Prolog: <?xml version="1.0" encoding="UTF-8"?>
  • Nested elements for hierarchy
  • Comments: <!-- This is a comment -->

Summary

XML is a flexible way to represent structured data in a standardized format. Its extensibility, readability, and platform independence make it widely used in data exchange, configuration files, and web services.

Rules.dtd
What is DTD?

What is DTD?

DTD (Document Type Definition) is a set of rules that defines the structure, elements, and attributes of an XML document. It ensures that an XML file is valid and follows the expected format.

Key Features of DTD

  • Specifies allowed elements and their hierarchy
  • Defines attributes for elements and their types
  • Ensures XML documents are consistent and valid
  • Can be declared internally or externally

Why Use DTD?

  • To enforce a standard structure for XML documents
  • To validate XML data before processing
  • To ensure compatibility between systems exchanging XML

Types of DTD

  • Internal DTD: Defined inside the XML document
  • External DTD: Stored in a separate file and linked to the XML document

Example of DTD

Internal DTD:

<?xml version="1.0"?>
<!DOCTYPE note [
  <!ELEMENT note (to,from,heading,body)>
  <!ELEMENT to (#PCDATA)>
  <!ELEMENT from (#PCDATA)>
  <!ELEMENT heading (#PCDATA)>
  <!ELEMENT body (#PCDATA)>
]>

<note>
  <to>Alice</to>
  <from>Bob</from>
  <heading>Reminder</heading>
  <body>Don't forget our meeting.</body>
</note>

Summary

DTD defines the structure and rules of an XML document, making it easier to validate, maintain, and exchange data reliably between systems. It is an essential part of working with XML in a structured and consistent way.

Style.xsl
What is XSL?

What is XSL?

XSL (eXtensible Stylesheet Language) is a language used to transform and style XML documents. It defines how XML data should be displayed or converted into other formats like HTML, plain text, or even another XML structure.

Key Components of XSL

  • XSLT (XSL Transformations): Used to transform XML data into other formats.
  • XPath: A language for navigating and selecting nodes in an XML document.
  • Formatting Objects (XSL-FO): Defines how XML data is formatted for output, e.g., PDF.

Why Use XSL?

  • To separate data (XML) from presentation (HTML, PDF, etc.)
  • To transform XML data into different formats for web or print
  • To filter, sort, or rearrange XML content dynamically
  • To create consistent styling for XML-based documents

Basic Example of XSLT

<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
  <xsl:template match="/">
    <html>
      <body>
        <h2>Book List</h2>
        <ul>
          <xsl:for-each select="books/book">
            <li><xsl:value-of select="title"/> by <xsl:value-of select="author"/></li>
          </xsl:for-each>
        </ul>
      </body>
    </html>
  </xsl:template>
</xsl:stylesheet>

Summary

XSL allows developers to transform and style XML data for different outputs. By combining XSLT, XPath, and XSL-FO, it provides powerful tools to separate content from presentation, making XML more versatile for web and document processing.

What is JSON?

JSON (JavaScript Object Notation) is a lightweight data-interchange format that is easy for humans to read and write, and easy for machines to parse and generate. It is commonly used for transmitting data between a server and a web application.

Key Features of JSON

  • Text-based and human-readable
  • Language-independent, but based on JavaScript object syntax
  • Lightweight compared to XML
  • Supports hierarchical (nested) data structures
  • Widely used for APIs and web services

Why Use JSON?

  • Easy to read and write for humans
  • Easy to parse and generate by machines
  • Ideal for client-server communication (AJAX, REST APIs)
  • Supports arrays, objects, strings, numbers, booleans, and null

JSON Syntax

{
  "name": "Alice",
  "age": 30,
  "isStudent": false,
  "skills": ["HTML", "CSS", "JavaScript"],
  "address": {
    "city": "New York",
    "zip": "10001"
  }
}

Summary

JSON is a simple and efficient format for exchanging data between systems. Its readability and lightweight structure make it the preferred choice for modern web applications and APIs.

DOM_Tree.js

What is the DOM?

DOM (Document Object Model) is a programming interface for HTML and XML documents. It represents the page as a structured tree of objects (nodes), allowing scripts like JavaScript to access, modify, and manipulate the content, structure, and style of web pages dynamically.

Key Features of the DOM

  • Represents a web page as a tree of nodes: elements, attributes, text, etc.
  • Enables dynamic updates of content and structure
  • Provides methods to traverse, add, remove, or modify nodes
  • Language-independent, but most commonly used with JavaScript in browsers
  • Standardized by the W3C

Why Use the DOM?

  • To dynamically change HTML content without reloading the page
  • To respond to user interactions (clicks, form input, etc.)
  • To manipulate styles and attributes dynamically
  • To create interactive and responsive web applications

Basic Example of DOM Manipulation (JavaScript)

<!DOCTYPE html>
<html>
<body>
  <p id="demo">Hello, World!</p>
  <button onclick="changeText()">Click Me</button>

  <script>
    function changeText() {
      document.getElementById("demo").innerHTML = "Hello, DOM!";
    }
  </script>
</body>
</html>

Summary

The DOM provides a bridge between HTML/XML documents and programming languages. By using the DOM, developers can create dynamic, interactive, and responsive web pages that react to user input and events in real time.

M04_Notes.doc

JavaScript and HTML Integration

JavaScript (JS) is a scripting language used to add interactivity and dynamic behavior to web pages. To make JS work, it must be connected to HTML markup. There are several ways to integrate JavaScript into an HTML document.

1. Inline JavaScript

JavaScript can be added directly to an HTML element using event attributes such as onclick. 

<button onclick="alert('Hello!')">Click Me</button>

2. Internal JavaScript

JavaScript can be written inside a <script> tag within the HTML file, usually in the <head> or at the end of the <body>. 

<!DOCTYPE html>
<html>
<head>
  <script>
    function greet() {
      alert('Hello from internal JS!');
    }
  </script>
</head>
<body>
  <button onclick="greet()">Greet</button>
</body>
</html>

3. External JavaScript

JavaScript can be placed in a separate .js file and linked using the src attribute of the <script> tag. 

<!-- index.html -->
<!DOCTYPE html>
<html>
<head>
  <script src="script.js"></script>
</head>
<body>
  <button onclick="sayHello()">Say Hello</button>
</body>
</html>

<!-- script.js -->
function sayHello() {
  alert('Hello from external JS!');
}

Best Practices

  • Place scripts at the end of <body> for faster page loading.
  • Keep JavaScript separate in external files for maintainability.
  • Use unobtrusive JavaScript (add event listeners in JS, not inline in HTML).
  • Use defer or async attributes to control script loading.

Summary

Integrating JavaScript with HTML allows developers to create interactive, dynamic, and responsive web pages. Inline, internal, and external scripts all have their uses, but separating code into external files and using event listeners is the modern best practice.

Basics.js

JS Basics

JavaScript (JS) is a programming language used to add logic and interactivity to web pages. Its core building blocks include variables, conditionals, and loops.

1. Variables

Variables store data values. In modern JavaScript, use let or const instead of var. 

let name = "Alice";    // can be changed
const pi = 3.14159;    // constant value

2. Conditionals

Conditionals control the flow of code using if, else if, and else. 

let score = 85;
if (score >= 90) {
  console.log("A grade");
} else if (score >= 75) {
  console.log("B grade");
} else {
  console.log("C grade");
}

3. Loops

Loops allow repeated execution of code blocks. Common types include for and while. 

// For loop
for (let i = 0; i < 5; i++) {
  console.log(i);
}

// While loop
let j = 0;
while (j < 5) {
  console.log(j);
  j++;
}

Summary

Understanding variables, conditionals, and loops is essential to writing functional JavaScript. These basics let you store data, make decisions, and repeat actions dynamically in your web pages.

Func.js

Functional JS

Functional JavaScript emphasizes using functions to process data in a clean and predictable way. Key concepts include arrow functions, map, filter, and reduce.

1. Arrow Functions

Arrow functions are a concise way to write functions. 

// Traditional function
function add(a, b) {
  return a + b;
}

// Arrow function
const addArrow = (a, b) => a + b;

console.log(addArrow(2, 3)); // 5

2. Map

map() creates a new array by applying a function to each element of an existing array. 

const numbers = [1, 2, 3, 4];
const squares = numbers.map(num => num * num);
console.log(squares); // [1, 4, 9, 16]

3. Filter

filter() creates a new array with elements that pass a test defined in a function. 

const ages = [12, 18, 25, 30];
const adults = ages.filter(age => age >= 18);
console.log(adults); // [18, 25, 30]

4. Reduce

reduce() reduces an array to a single value by applying a function cumulatively. 

const numbers = [1, 2, 3, 4];
const sum = numbers.reduce((total, num) => total + num, 0);
console.log(sum); // 10

Summary

Functional JavaScript techniques like arrow functions, map, filter, and reduce allow you to write cleaner, more declarative code for processing arrays and data. They are essential for modern JS development.
