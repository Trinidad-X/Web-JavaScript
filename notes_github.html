02. Core Concepts: Git vs. GitHub

Many people confuse the two. Here is the distinction:

Git (The Engine) GitHub (The Host)
  • Installed on your local computer.
  • Tracks changes in files (Version Control).
  • Works offline.
  • Command line tool (CLI).
  • A website (Cloud Service).
  • Stores your Git repositories online.
  • Adds collaboration (Pull Requests, Issues).
  • Visual Interface (GUI).

04. Dictionary (Cheat Sheet)
Repository (Repo)
The folder containing your project and history.
Commit
A "Save Point". A snapshot of your code at a specific time.
Pull Request (PR)
Asking to merge your branch into the main branch. This is where code review happens.
Fork
Copying someone else's repo to your account so you can modify it.
